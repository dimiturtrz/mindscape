# THINGS-EEG1 (ds003825, Grootswagers et al. 2022) — data-adapter spec

**Date:** 2026-07-08 · **Why:** write a `core/data/eeg/things_eeg1.py` adapter mirroring `things_eeg2.py`, to enable a
**cross-dataset concept-level zero-shot retrieval** test (train on one THINGS-EEG set, eval on the other). All
specs below are verified against the **primary source**: Grootswagers, Zhou, Robinson, Hebart & Carlson, *Scientific
Data* **9**:3 (2022), doi:10.1038/s41597-021-01102-7, and the live BIDS files on OpenNeuro `ds003825` (v1.2.0). Where
a value is inferred (not stated), it is flagged `[inferred]`.

---

## 1. BIDS layout of ds003825

**Snapshot `1.2.0`, 44,258,973,478 bytes (~41.2 GiB / 44.3 GB), 50 subjects, 257 files, modality `eeg`, task `rsvp`**
(verified via OpenNeuro GraphQL `dataset(id:"ds003825").latestSnapshot`).

```
ds003825/
  dataset_description.json, README, CHANGES, participants.tsv, participants.json
  task-rsvp_eeg.json                      # shared EEG sidecar (acquisition params)
  task-rsvp_events.json                   # shared events-column dictionary (the KEY file — see §3)
  sub-01/ .. sub-50/
    eeg/
      sub-XX_task-rsvp_eeg.vhdr           # BrainVision header  ── RAW is BrainVision, NOT EEGLAB
      sub-XX_task-rsvp_eeg.eeg            # BrainVision binary
      sub-XX_task-rsvp_eeg.vmrk           # BrainVision markers (E1 stim-onset / E2 offset / E3 seq-start)
      sub-XX_task-rsvp_events.tsv         # ~3.3 MB text, one row per stimulus (concept name inline!)
      sub-XX_task-rsvp_events.csv         # git-annex pointer (ignore; the .tsv is the real one)
      sub-XX_task-rsvp_eeg.json           # per-subject EEG sidecar
  derivatives/
    eeglab/ sub-XX_task-rsvp_continuous.set + .fdt   # authors' preprocessed continuous EEGLAB (≈47 subj present)
    RDM/                                              # group-average representational dissimilarity matrices
```

- **Raw format = BrainVision triplet** `.vhdr/.eeg/.vmrk` (BrainVision ActiChamp amplifier). Read with
  `mne.io.read_raw_brainvision(...)` or `mne_bids.read_raw_bids(...)`.
- **No sessions, no runs, no `ses-*`/`run-*`** — exactly **one recording per subject** (`task-rsvp`, ~1 h, ~25k
  stimulus events). This is *flatter* than EEG2 (which has `ses-*/raw_eeg_{training,test}.npy` per subject).
- **Preprocessed EEGLAB is provided** in `derivatives/eeglab/` (`_continuous.set/.fdt`) — but it is the authors'
  minimal pipeline (§4); we preprocess off the **raw BrainVision** for parity with EEG2.
- License: **CC-BY-4.0** (data); metadata **CC0**.

## 2. Paradigm (RSVP target-detection)

- **50 subjects** (36 F / 14 M, mean age 20.44 ± 2.72, range 17–30). 4 flagged for potential exclusion (poor
  signal/equipment failure) but **included** — marked in `participants.tsv`.
- **EEG:** BrainVision **ActiChamp**, **64 electrodes**, international **10–10** layout, **online reference = Cz**,
  digitised at **1000 Hz**, resolution 0.0488281 µV. (No EOG/EMG channels — the paper notes frontal EEG could be used
  for eye-movement correction.)
- **RSVP stream:** images at **10 Hz, 50 % duty cycle → 50 ms on + 50 ms blank, SOA = 100 ms.** Viewing distance
  ~57 cm, images subtend ~10°. A 0.5° bullseye fixation overlaid on every image.
- **Stimuli = THINGS.** 1,854 concepts × (first 12 images/concept) = **22,248 unique images**, shown **once each** in
  the main session, split into **72 sequences of 309 stimuli** (concepts never repeat within a sequence).
- **Validation block (after main):** the **same 200 THINGS validation images** (listed in `test_images.csv`),
  each repeated **12 times**, in **12 sequences of 200** (random order) = 2,400 trials. **Total = 84 sequences,
  ~24,648 stimulus events + ~25k with targets** in one ~1 h session.
- **Task (orthogonal):** 2–5 random **target events per sequence** where the bullseye turns **red for 100 ms** →
  button press. Marked via `istarget`/`response`/`correct` columns. Not stimulus-locked to any concept; a decoder
  ignores/excludes them.
- **Triggers (in `.vmrk`):** `E3` = sequence start, `E1` = **every stimulus onset**, `E2` = stimulus offset.
- Programmed in Python 3.7 / PsychoPy 3.0.5.

## 3. Trial → stimulus → THINGS-concept mapping  (THE load-bearing part)

**The THINGS concept name is written directly into `events.tsv`** — no separate lookup table needed (contrast EEG2,
which needs `image_metadata.npy`). Columns (from `task-rsvp_events.json`, verified against `sub-01`'s rows):

| column | meaning |
|---|---|
| `onset` | event onset — value ~10724, step +100 per stimulus, `duration`=50 → **units are ms (= samples @ 1000 Hz), relative to recording start** `[inferred; verify vs raw.n_times]` |
| `duration` | 50 (ms) |
| **`object`** | **THINGS concept NAME**, e.g. `carousel`, `piano`, `birdbath`, `stove2` — **the cross-dataset key** |
| **`objectnumber`** | concept id **0–1853** (THINGS order). **Alphabetical** — verified: birdbath 129 < calf1 241 < carousel 275 < drum 522 < piano 1165 < stove2 1564 < tent 1643 < turntable 1735 |
| **`stim`** | stimulus path, e.g. `stimuli\carousel\carousel_11s.jpg` (Windows separators; concept folder + `concept_NN{s,n}.jpg`) |
| `stimname` | bare filename, e.g. `carousel_11s.jpg` |
| `stimnumber` | within-concept image index **0–11** |
| `isteststim` | 1 if from the 200-image validation block |
| `teststimnumber` | validation-image index **0–199** (else n/a) |
| `sequencenumber`, `withinsequencenumber`, `presentationnumber`, `blocksequencenumber` | ordering/structure |
| `istarget`, `response`, `rt`, `correct` | target-detection behaviour (exclude for decoding) |
| `time_stimon`, `time_stimoff`, `stimdur` | high-precision onset/offset (s from **experiment** start, ≠ recording start) |

So each epoch → concept name via `object` and image file via `stimname` **directly from the row**. To align to EEG2's
concept index: EEG2 encodes concept as `"00001_aardvark"` (5-digit, 1-based, alphabetical) → index 0; EEG1
`objectnumber` is 0-based alphabetical over the same 1,854 THINGS concepts, so `EEG1.objectnumber == EEG2.concept_idx`
**and** the `object` string matches EEG2's name suffix. **Bridge on the concept NAME** (robust to any index drift).

## 4. Preprocessing (authors) & raw availability

Authors' **technical-validation** pipeline (MATLAB R2020b + EEGLAB v14.0.0b), applied to the `derivatives/eeglab`
data only — deliberately minimal:
- Hamming-windowed FIR filter, **0.1 Hz high-pass, 100 Hz low-pass**.
- **Re-reference to average** (raw is online-Cz).
- **Downsample 1000 → 250 Hz.**
- **Epoch [−100, 1000] ms** relative to stimulus onset.
- **No baseline correction, no channel selection** — they explicitly leave baseline correction / eye-movement
  correction to the user. Channel voltage at each time point is the MVPA input.

**Raw is fully available** (BrainVision, 1000 Hz, all 64 ch) → we do **our own preprocessing off the raw** (mirroring
EEG2), so the two adapters share an identical downstream pipeline (bandpass → z-score → resample).

## 7. Adapter sketch — `core/data/eeg/things_eeg1.py` (mirrors `things_eeg2.py`)

Data lives at `<data>/raw/things_eeg1/` (BIDS tree). One subject → `(EEG [n,64,t] float32, concept-name[n],
img_file[n])`, same signature as `things_eeg2.get_epochs` but concept/file come from `events.tsv` rows, not a stim
trigger channel + `image_metadata.npy`.

```python
class ThingsEeg1EpochCfg(BaseModel):
    tmin: float = 0.0; tmax: float = 1.0      # window (s) around stim onset — matches EEG2 default [0,1]
    resample: float = 250.0                   # Hz (EEG1 raw is 1000; EEG2 default 250 too)
    fmin: float | None = None; fmax: float | None = None
    include_validation: bool = False          # False -> main 22,248 only; True -> +200x12 validation block
    drop_targets: bool = True                 # exclude fixation-colour target trials (istarget==1)

def _subject_epochs(sub_dir, cfg):
    raw = mne.io.read_raw_brainvision(sub_dir/"eeg"/f"{sub}_task-rsvp_eeg.vhdr", preload=True)  # 64ch @1000Hz
    ev  = pl.read_csv(sub_dir/"eeg"/f"{sub}_task-rsvp_events.tsv", separator="\t")               # one row/stim
    fs  = raw.info["sfreq"]                                                                       # 1000.0
    eeg = raw.get_data()                                                                          # [64, T] volts
    if cfg.fmin or cfg.fmax: eeg = bandpass(eeg, cfg.fmin or 0.1, cfg.fmax or fs/2-1, fs)

    # onset -> sample. events.tsv onset is ms(=samples @1000Hz) from recording start [VERIFY: max(onset)<=T].
    # ROBUST alt: read E1 markers from the .vmrk (authoritative sample indices), zip 1:1 with ev rows in order.
    onset = (ev["onset"].to_numpy() * fs / 1000.0).round().astype(int)
    keep  = pl.Series(range(len(ev)))
    if cfg.drop_targets:        keep = keep.filter(ev["istarget"] == 0)     # or keep targets, they still have a concept
    if not cfg.include_validation: keep = keep.filter(ev["isteststim"] == 0)
    onset, rows = onset[keep], ev[keep]

    s0, s1 = round(cfg.tmin*fs), round(cfg.tmax*fs)
    ok = (onset+s0 >= 0) & (onset+s1 <= eeg.shape[1]); onset, rows = onset[ok], rows[ok]
    epochs = np.stack([eeg[:, a+s0:a+s1] for a in onset]).astype(np.float32)   # [n,64,t]
    epochs = (epochs - epochs.mean(2,keepdims=True)) / (epochs.std(2,keepdims=True)+1e-7)  # per-ch z (EEG2 parity)
    if cfg.resample != fs:
        epochs = _resample(epochs, round(epochs.shape[2]*cfg.resample/fs), axis=2).astype(np.float32)
    return epochs, rows["object"].to_numpy(), rows["stimname"].to_numpy()      # concept NAME, image file
```

`get_epochs(subjects, cfg)` then concatenates subjects → `(eeg[n,64,t], concept_name[n], img_file[n],
meta{subject})`. For cross-dataset retrieval, map `concept_name` → the shared CLIP/semantic target the same way
`neuroscan/tasks/visual/clip_targets.py` does for EEG2 (join on the concept string).

## 8. Discrepancies vs THINGS-EEG2 the cross-dataset test MUST reconcile

| axis | **THINGS-EEG1** (ds003825) | **THINGS-EEG2** (Gifford, OSF) | reconciliation |
|---|---|---|---|
| **channels** | **64** EEG (10-10, online ref Cz) | **63** EEG (+1 stim trigger ch) | intersect to common channel names / re-montage; can't assume identical sets |
| **montage** | ActiChamp 10-10, Cz-referenced raw | actiCAP 63, avg/other | re-reference EEG1 to average to match; align by electrode label |
| **sampling rate** | 1000 Hz raw | 1000 Hz raw | both → resample 250 Hz (shared) — OK |
| **SOA / rate** | **100 ms (50 on/50 blank), 10 Hz** | **200 ms (100 on/100 blank), 5 Hz** | EEG1 faster → stronger forward/backward masking, shorter clean window; epoch window & expected SNR differ |
| **repetition/SNR** | each of 22,248 imgs shown **once** (broad, low per-image SNR); 200 val × 12 | train 4×, **test 80×** (deep, high SNR) | EEG1 can't trial-average per image (except the 200 val ×12); retrieval must be concept-level, not image-level |
| **concept coverage** | **all 1,854** concepts × 12 exemplars | 1,654 train + 200 test = 1,854 | EEG2's concept set ⊆ EEG1's; near-full concept overlap for zero-shot |
| **concept encoding** | `object` name **inline in events.tsv** | 1-based stim code → `image_metadata.npy` | different parse paths; both resolve to the same THINGS names |
| **image files** | THINGS first-12/concept, `concept_NN{s,n}.jpg` | EEG2's own train/test file split | **image-file identity NOT guaranteed** except possibly the 200 val/test set → **bridge on concept NAME, not file** |
| **structure** | 1 session / 1 run / subject | `ses-*/raw_eeg_{training,test}.npy` | flatter loader; no train/test split baked in (we choose main vs validation) |
| **on-disk** | BrainVision `.vhdr/.eeg/.vmrk` + BIDS `events.tsv` | per-session `.npy` dicts | EEG1 needs MNE + tsv parse; EEG2 needs npy + trigger-channel onset detection |

**Net:** the honest cross-dataset test is **concept-level zero-shot retrieval** (train EEG2 → eval EEG1 subjects, and
vice-versa), bridged on the THINGS **concept name**. The channel-count/montage mismatch (64 vs 63) and the SOA/SNR gap
(10 Hz single-shot vs 5 Hz 80× repeats) are the two axes that will drive the generalization drop — which is exactly
the effect our contribution measures, not hides.
